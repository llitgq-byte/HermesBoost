# Finance

**🇨🇳 中文** · [English](README.md)

> 股票研究与分析——从盘前到盘后。

基于 Hermes 技能构建的个性化股票研究工作流：盘前简报、盘中追踪和盘后复盘。每个工作流都可根据个人持仓和投资策略进行定制，将原始市场数据转化为可执行的分析背景。

## 本类别的功能

金融流水线自动化了大多数投资者手动完成的研究循环——开盘前扫描隔夜新闻、盘中监控持仓、收盘后复盘决策。每个阶段都是一个独立的 Hermes 技能，但组合在一起便形成一个连续的工作流，前一阶段的输出直接作为下一阶段的输入。

目标不是预测市场或生成交易信号，而是**减少噪音，呈现关键信息**——让你坐下来做决策时，前期工作已经完成。

## 三个阶段

### 1. 盘前简报

开盘前，简报汇总市场休市期间发生的所有事件，并将其提炼为结构化摘要。

**涵盖内容：**
- **隔夜异动** — 盘后及国际市场的显著价格波动
- **头条与催化剂** — 财报发布、宏观数据、行业新闻、分析师上调/下调评级
- **关键价位** — 基于近期价格走势和技术形态推导的支撑/阻力区间
- **开盘倾向** — 基于盘前指标（期货、跳空缺口、成交量）的方向性判断

**为什么重要：** 大部分损失（和机会）都发生在开盘前 30 分钟。带着分析背景而不是仓促应对开始新的一天，会改变后续每个决策的质量。

### 2. 盘中追踪

在交易时段，追踪器监控持仓和市场状况，标记需要关注的事件。

**监控内容：**
- **成交量异常** — 突然激增，往往预示或确认价格波动
- **动能转变** — 趋势加速、反转信号、衰竭形态
- **相关性断裂** — 持仓与其所在板块或大盘走势出现背离
- **风险阈值** — 触发预设止损或止盈意识的价格水平

**为什么重要：** 你无法盯住每一只持仓的每一个跳动。追踪器充当过滤器——无事发生时保持安静，有异常时及时提醒。

### 3. 盘后复盘

收盘后，复盘还原当天的交易，并对照实际走势评估决策质量。

**审查内容：**
- **执行质量** — 是否在合理价位进出，还是在追涨杀跌？
- **催化剂准确性** — 盘前逻辑是否兑现，还是有意外事件改变了叙事？
- **流程评估** — 即使结果不理想，决策过程本身是否合理？
- **行动事项** — 为下一交易时段制定具体调整方案（仓位管理、时机选择、自选股变更）

**为什么重要：** 进步来自复盘流程，而非结果。一笔好的交易仍能教会你一些东西；一笔因正确流程而导致的亏损交易则不会——复盘能帮你区分两者。

## 数据来源

所有数据均来源于**公开渠道**：

- **市场数据** — 来自公共 API 的实时和延迟行情
- **新闻与头条** — 聚合自公开财经新闻源
- **基本面数据** — 通过公开渠道获取的公司公告和分析师报告
- **技术指标** — 基于原始 OHLCV（开高低收量）数据计算

不使用专有数据，不涉及内幕信息，不依赖付费 Alpha 信号。价值不在于独占数据，而在于数据的筛选、组织和呈现方式。

## 适用人群

面向**管理个人投资组合的个人投资者**构建——不是追逐每一个波动的日内交易者，也不是配备 Bloomberg 终端的机构团队，而是希望获得结构化研究流程、又不想从零开始搭建的人。

这些工作流带有特定倾向：它们反映了一种特定的投资风格和优先级。如果你的方法不同，可以将这些技能作为模板使用——逻辑透明，结构模块化。

## 设计原则

- **无股票代码、无持仓信息** — 技能通用且可配置；个人持仓存储在本地，不会进入代码仓库
- **流程优于预测** — 重点在于准备和复盘，而非生成买卖信号
- **精简且可读** — 每个技能专注做好一件事；不使用臃肿的多用途工具
- **坦诚面对局限性** — 这些工作流有助于建立研究纪律，但不能替代判断
