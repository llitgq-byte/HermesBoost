# Auxiliary

**🇨🇳 中文** · [English](README.md)

---

> 系统级保护，而非能力提升 —— 减少重复、捕获错误。

为 Hermes Agent 添加**保护层**的技能（Skills）与钩子（Hooks）—— 让使用更顺畅，减少重复犯错。目的不是让 Agent 更聪明，而是让它在实际使用中更可靠。

## 什么适合放在这里

此分类面向解决**系统级摩擦点**的模块 —— 即无论你执行什么任务，都会跨会话反复出现的问题。

| 类型 | 说明 |
|------|------|
| **技能（Skills）** | 可复用的工作流，将可重复的流程编码其中，确保同类任务每次执行方式一致。 |
| **钩子（Hooks）** | 事件驱动的拦截器，在系统操作前后触发，用于添加验证、安全检查或自动修正。 |

一个模块归入此处需满足两个条件：

1. **具有横切性** —— 无论你让 Hermes 做什么，它都有用。
2. **具有保护性** —— 它旨在防止某类错误，而非启用新能力。

## 模块

### [memory-file-guard](memory-file-guard/README.md)

Hermes Agent 插件，拦截对记忆文件和用户文件的写入操作，在变更提交前要求用户明确批准。防止 AI 静默覆写持久化记忆 —— 这是 Agent 系统中破坏性最强的一类错误。

**简而言之：** 每次对 `MEMORY.md` 和 `USER.md` 的写入都会被阻断，直到你确认同意。

## 适用人群

任何希望**减少静默回退**的 Hermes Agent 用户。如果你曾经遇到过 Agent 在会话中途损坏记忆文件的情况，这些模块就是为你而存在的。

## 贡献模式

这里的每个模块都诞生于真实的摩擦点，而非理论上的需求。新模块以同样的方式添加：发现问题、构建修复方案、将其泛化。

标准很简单：如果它能帮你避免两次犯同样的错误，它就属于这里。
